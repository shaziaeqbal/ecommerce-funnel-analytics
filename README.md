## What's in here

The data is fully synthetic (50k sessions, ~11k transactions, $684k revenue) but structured to match real GA export patterns — channels, devices, countries, bounce rates, session durations. The funnel sits at 65% view rate, 26% cart rate, 8.8% purchase rate, with a 67% cart abandonment rate, which is close to industry benchmarks.

From there, a dbt pipeline cleans and models the data into staging views and business-ready mart tables. The ML model (GradientBoosting) trains on the `mart_cart_abandonment` table and scores each cart session as Low / Medium / High abandonment risk. The Streamlit dashboard ties everything together — funnel visualization, channel revenue breakdown, ML risk distribution, and a revenue recovery business case.

## Quickstart

```bash
git clone https://github.com/shaziaeqbal/ecommerce-funnel.git
cd ecommerce-funnel

pip install -r requirements.txt

# generates data, runs dbt, trains model
python setup.py

# run the dashboard
python -m streamlit run dashboard/app.py
```

> **Note:** Use `python -m streamlit` rather than `streamlit` directly — this ensures the dashboard resolves the `ml` module correctly regardless of how your Python path is configured.

The `.duckdb` file is already committed so you can skip `generate_data.py` and go straight to `dbt run` + model training if you want.

## dbt models

Three-layer structure — staging views type-cast the raw tables, intermediate models compute funnel rates and channel attribution, mart tables are materialized and consumed by the dashboard and ML pipeline.

`mart_revenue_impact` estimates lost revenue per channel and models 10% / 20% recovery scenarios based on average order value.

---

## ML model

**Algorithm:** GradientBoostingClassifier (scikit-learn)
**Target:** binary — abandoned (added to cart, did not purchase)
**Features:** channel, device, country, page views, session duration, bounce flag, reached checkout flag
**Output:** abandonment probability → bucketed into Low / Medium / High risk
**ROC-AUC:** ~0.80

The trained model is saved to `ml/abandonment_model.joblib`. `predict.py` exposes a `score_sessions(df)` function that handles unseen labels gracefully and returns a DataFrame with the `abandonment_probability` column appended.

## Dashboard

Five sections, filterable by channel and device from the sidebar:

- **Funnel** — waterfall from product view through purchase with step-level drop-off
- **Channel revenue** — bar chart of total revenue and orders per channel
- **Device conversion** — pie chart of purchase rate by device
- **ML risk scores** — distribution of Low / Medium / High risk across cart sessions, with a scrollable scored table
- **Revenue recovery** — business case table per channel with 10% and 20% recovery estimates

## Business case numbers

At the current 67% cart abandonment rate:

| Scenario | Est. recovery |
|---|---|
| 10% of abandoned carts recovered | ~$45,000 |
| 20% of abandoned carts recovered | ~$91,000 |
| Cost of targeted campaign | ~$3,000 |
| ROI | 15–30× |

Targeting ML-scored high-risk sessions (score > 0.7) with a personalised email or exit-intent offer is the intervention modelled here.
